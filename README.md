# 🎨 Kanban Task Manager — Frontend

A state-of-the-art Kanban task management client built with **React 19**, **TypeScript**, **Vite**, and **Apollo Client**.

Features fluid drag-and-drop boards, paginated task tables, role-based visual elements, and a high-performance shadcn/ui design system with full dark-mode support.

---

## 🚀 Features

* **Session Restoration & Context**: `AuthProvider` runs a `Me` query on startup to automatically restore authenticated context.
* **Role-Based Protected Routing**: `ProtectedRoute` gates views (e.g. `/users` requires `ADMIN` role).
* **Responsive Drag-and-Drop Columns**: Reorderable status lists powered by `@dnd-kit/core` and `@dnd-kit/sortable`.
* **State Management**: Built on Apollo Client's in-memory caching and reactive variables.
* **Unified Theming System**: Dark mode configuration powered by Tailwind CSS v4 custom properties.
* **Reusable shadcn/ui Components**: Compact primitives (`Card`, `Dialog`, `Input`, `Badge`, `Pagination`, `ConfirmationDialog`, `PageHeader`, `StatePanel`).

## ⚙️ Setup & Development

### 1. Install Dependencies
Navigate to [frontend](/frontend) and run:
```bash
npm install
```

### 2. Configure Environment variables
Copy [.env.example](/frontend/.env.example) to `.env`:
```bash
cp .env.example .env
```
Confirm your backend GraphQL gateway endpoint:
```env
VITE_GRAPHQL_URL=http://localhost:4000/graphql
```

### 3. Generate Type Bindings
Run the GraphQL codegen compiler to build types matching the backend schema:
```bash
npm run codegen
```
*Note: Make sure your GraphQL backend is running before generating types.*

## 🎨 Theme & Theming Tokens

Styling utilizes **Tailwind CSS v4** config with variables defined under [src/index.css](file:///c:/Users/minph/Documents/kanban-task-manager/frontend/src/index.css):

```css
@theme {
  --font-sans: 'Geist Variable', sans-serif;
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-primary: var(--primary);
  /* Mapped colors coordinate directly with light/dark tailwind theme configs */
}
```

Dark mode is controlled via standard body class injection (`.dark`).
